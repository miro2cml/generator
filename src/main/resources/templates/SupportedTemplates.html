Templates

miro2cml supports three templates: User Stories, EventStorming and Bounded Context Canvas.

Precondition

To be able to follow this user guide, we expect you to be familiar with Miro and the concepts of Tactic and Strategic Domain Driven Design.

Mapping-Heuristics

The mapping-heuristics are described in the sections bellow. The mapping-heuristic explain in details, how the mapping is done by miro2cml. For step by step descriptions for the miro2cml visit the page Tutorials.

Which inputs are converted?

The User Story Map contains only Cards. Therefore just Cards will be converted. All other Miro elements are ignored. The Card which are yellow and blue are ignored. Because they illustrate the Tasks and Activites in the User Story Map. The title of the Cards are relevant for the mapping. The description and the other fields of the Card will be ignored.

Frequently Asked Questions (FAQ)

Why are my Cards not converted?

  • The Cards are yellow or blue.
  • They have not the correct text format. Check if you use the correct format like introduced in the tutorial.
  • The User Stories are not in the title defined of the Miro Cards.

How is the title of the User Story generated?

The title is the verb concatenated with the entity of the User Story.

Why there are some numbers added to the title of the User Story?

The Context Mapper expect unique titles for the User Stories. If there are exists equal titles the miro2cml converter add an 1 at the end of the name to make sure that the conversion produce a correct CML File.

Why some verbs have quotations mark at the beginning and at the end and some not?

The Context Mapper knows the CRUD verbs (create, read, update, delete). Therefore they do not need quotation marks. All other verbs are between the quotations marks to mark them as a string.

User Guide for UserStories
  • Step One: Create the MiroBoard

    First, select the template User Story Map Framework in Miro as shown in the picture bellow.

    Then you fill in the User Stories. The miro2cml converter only uses the title of the Cards, you can use the rest of the Cards like you want. In the picture below, an empty and a filled out Card are shown.

    Make sure that you use one of these formats for the User Stories:

    • As a < actor > I want to < verb > a < entity > so that < benefit >.
    • As an < actor > I want to < verb > an < entity > so that < benefit >.
    • As a < actor > I want to < verb > an < entity > so that < benefit >.
    • As an < actor > I want to < verb > a < entity > so that < benefit >.

    The yellow and blue Cards (in the picture red bordered) are not converted as User Stories. They are used as Tasks and Activities in the Template and are therefore not relevant for the mapping.

  • Step Two: Convert the Board

    After you have created your board, you can convert it with the miro2cml Tool. Choose your board and the Board Type: User Stories. For the example in the figure one the following output will be generated:

    
                        UserStory createaccount {
                            As a "User"
                            I want to "create" an "account"
                            so that "I could register".
                        }

    The name of the User Story is composed of the verb and the entity. The rest is the same as your input

  • Step Three: Use the CML output

    The CML File with your User Stories is a good start for design work with the Context Mapper Tool. Here is a Link to a step by step tutorial for further design work.

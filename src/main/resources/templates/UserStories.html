User Guide for UserStories
  • Step One: create the MiroBoard

    First you select in Miro the template User Story Map Framework. Shown in the picture bellow. Then you could fill in the User Stories. The miro2cml converter note only the title of the Cards, you could use the rest of the Cards like you want. In the picture is an empty and a filled out Card to see. Make sure that you use, one of these formats for the User Stories:

    • As a < actor > I want to < verb > a < entity > so that < benefit >.
    • As an < actor > I want to < verb > an < entity > so that < benefit >.
    • As a < actor > I want to < verb > an < entity > so that < benefit >.
    • As an < actor > I want to < verb > a < entity > so that < benefit >.

    The yellow and blue Cards are not converted, in the picture red bordered. They are used as Tasks and Activities in the Template and therefore are not relevant for the mapping.
  • Step Two: convert the Board
    After you have created your board, you could convert it with the miro2cml Tool. Choose your board and the Template: User Stories. For the example in the figure one the following output will be generated: 
    
                UserStory createaccount {
                    As a "User"
                    I want to "create" an "account"
                    so that "I could register".
                }

    The name of the User Story is composed of the verb and the entity. The rest is the same as your input

  • Step Three: use the CML ouput

    The CML File with your User Stories is a good start for design work with the Context Mapper Tool. Here is a Link to a step by step tutorial for further design work. You could skip the first step because you already have your Requirements defined. If you're new to the Context Mapper here a Link to how you get started with it.

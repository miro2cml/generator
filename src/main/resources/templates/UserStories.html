User Guide for UserStories
  • Step One: Create the MiroBoard

    First, select the template User Story Map Framework. The template can be found in the section "Chose a Template" in Miro. The preview of the template is shown in the figure.

    Next, you can create the User Stories, for instance by filling in text into the empty stories or by copy-pasting a story card from a working example. The miro2cml converter only uses the title of the cards, so you can use the rest of the Cards as you wish. In the picture below, an empty and a filled out card are shown.

    Make sure that you use one of these formats and characters for the User Stories:

    • As a < actor > I want to < verb > a < entity > so that < benefit >.
    • As an < actor > I want to < verb > an < entity > so that < benefit >.
    • As a < actor > I want to < verb > an < entity > so that < benefit >.
    • As an < actor > I want to < verb > a < entity > so that < benefit >.

    Objects Allowed Characters Java Regex
    < actor > upper and lowercase letters,
    numbers    		 [A-Za-z0-9]+
    < verb > leading lowercase letter followed by
    upper and lowercase letters,
    numbers    		 [a-z][a-zA-Z0-9]+
    < entity > upper and lowercase letters,
    numbers    		 [a-zA-Z0-9]+
    < benefit > upper and lowercase letters,
    numbers, point, comma, colon
    space, apostrophe    		 [A-Z,. ':a-z0-9]+

    The yellow and blue cards (in the picture, these cards have a red border) are not converted as User Stories. They are used as Tasks and Activities in the Template and are therefore not relevant for the mapping.

  • Step Two: Convert the Board

    After you have created your board, you can convert it with the miro2cml tool. Choose your board and the board type "User Stories". From the figure, for example, one the following outputs will be generated:

    
                        UserStory createaccount {
                            As a "User"
                            I want to "create" an "account"
                            so that "I could register".
                        }

    The name of the User Story is composed of the verb and the entity. The rest is the same as your input. Take a look at the mapping table for further information.

  • Step Three: Use the CML output

    The CML file with your User Stories is a good start for design work with the Context Mapper Tool. Follow this Link to a step by step tutorial for further design work. You can skip the first step in this tutorial, as you already have your requirements defined. If you're new to the Context Mapper here is a Link to the guide on how to Getting started.

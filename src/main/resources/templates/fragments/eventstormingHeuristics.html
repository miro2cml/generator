Frequently Asked Questions (FAQ)

Why is my board never converted without warnings?

The compiler is currently not able to recognise the end of an Event Storming model and therefore it always generates a warning in the last step.

Why are my Stickers not converted?

Check if one of the following applies to your board:
  • The color of the Sticker is not defined as a Event Storming element.
  • The Sticker do not have the correct format. Check if you use the correct format as introduced in the tutorial. The Command, Domain Event and Aggregate Stickers belonging to the same process step should be near each other.
  • The semantic model is not correct. Each process step must contain a Command, Domain Event and an Aggregate.

What does serializer error mean?

Something is wrong with your event storming model. Take another look at our tutorial and examples and make sure that you use a semantically correct model: Each process step must contain a Command, Domain Event and an Aggregate. If the model is correct, make sure that the stickers are recognized: Take a look at the log file for further information about the conversion. The stickers from the same step should be near each other. If they are grouped correctly, you should find a log entry which states group found.

Why is my board not recognized?

Our Converter for the Event Storming Boards expect you to define the colors for the Commands, Aggregates, Domain Events, User Role and Issue with Stickers, which have only the keywords written on it (see our examples). Make sure that all Stickers have a different color and verify if the keywords are spelled and formatted correctly.
